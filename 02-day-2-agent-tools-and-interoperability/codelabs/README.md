# 🧪 Day 2 Codelabs

This folder contains the hands-on Day 2 codelab work for **Agent Tools & Interoperability**.

Day 2 had two different practical angles. The first codelab focused on an agent using local development tools to build a working app. The second codelab focused on Antigravity using an MCP server to retrieve official Google developer documentation.

| Codelab | Status | Folder | What it demonstrates |
|---|---|---|---|
| Antigravity CLI hands-on app build | ✅ Completed | [`01-antigravity-cli/`](./01-antigravity-cli/) | Agent-assisted local app development, command execution, artifact generation, UI review, QA, Git checkpoints, and Render deployment. |
| Google Developer Knowledge MCP in Antigravity | ✅ Completed | [`02-google-developer-knowledge-mcp/`](./02-google-developer-knowledge-mcp/) | MCP setup, API-key based connection, tool discovery, permission review, and documentation-backed answers inside Antigravity. |

---

## 🧭 How these two codelabs fit together

The Day 2 theory work explains why tools and interoperability matter. These codelabs capture the practical side.

The first codelab answered:

> What does it feel like when an agent can work inside a local development folder and create a real project?

The second codelab answered:

> What does it feel like when an agent can call a connected documentation tool through MCP instead of relying only on model memory?

Together, they show why tool-connected agents need human review, scoped permissions, safe credential handling, and clear documentation.
