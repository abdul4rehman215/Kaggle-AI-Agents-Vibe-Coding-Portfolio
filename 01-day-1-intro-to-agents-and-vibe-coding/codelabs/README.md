This folder contains the hands-on codelab work completed for **Day 1 — Introduction to Agents and Vibe Coding**.

The goal of this folder is to keep the practical implementation work separate from the broader Day 1 notes and reflections. Each codelab has its own folder, source files, screenshots, implementation notes, and deployment/testing documentation.

## 🎯 Why this folder exists

Day 1 was not only about reading or watching course material.

It included practical work with two different agentic development environments:

1. **Google Antigravity** — used to explore agent planning, workspace behavior, code generation, review artifacts, and a custom code-review skill.
2. **Google AI Studio** — used to build a browser-only app from natural-language instructions, refine the behavior, export the source code, and test deployment.

Keeping both codelabs inside one folder makes the Day 1 implementation trail easier to review.

## 🧠 Learning focus

Across both codelabs, the main learning themes were:

- expressing software intent through natural language,
- reviewing AI-generated implementation plans and code,
- using agentic workflows for both creation and review,
- refining generated behavior through follow-up constraints,
- keeping source code and documentation reproducible,
- testing deployment while staying aware of cost and cleanup,
- documenting evidence instead of treating the work as a one-time experiment.

---

## 🔐 Safety note

The Day 1 app work was kept intentionally lightweight and low risk:

- no user accounts,
- no private data,
- no database,
- no server-side API requirement,
- no required Gemini API key in the cleaned portfolio copy,
- no paid external service dependency for the current Vercel demo.

The Cloud Run deployment was tested during the codelab and later unpublished for cost control. The current public link uses Vercel as a static frontend deployment.
