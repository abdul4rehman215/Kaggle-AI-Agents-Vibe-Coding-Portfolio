# 📚 Source Material Note - Day 3

This file documents what material was used for Day 3 notes and codelab documentation, and what was intentionally not committed to the repository.

---

## Source material used

The Day 3 theory notes were based on:

- Unit 3 podcast/video summary,
- Agent Skills whitepaper,
- NoteGPT-style podcast summary,
- NotebookLM study guide,
- NotebookLM quiz / Q&A review,
- NotebookLM explanation video,
- personal visual summaries created during revision.

The codelab documentation was based on:

- the official Antigravity Skills codelab,
- the official Agents CLI + ADK lifecycle codelab,
- terminal commands and outputs from the hands-on session,
- screenshots captured during Antigravity and ADK testing,
- curated source files from the local `my-first-project` workspace.

---

## What is committed

This folder includes:

- human-written theory notes,
- codelab notes,
- a compact glossary,
- a study-process summary,
- codelab READMEs,
- command logs,
- validation notes,
- troubleshooting notes,
- curated source snapshots,
- renamed screenshot evidence,
- two visual study assets.

The goal is to document what I understood and built, not to archive every raw intermediate file.

---

## What is intentionally not committed

The following are intentionally excluded:

- API keys,
- `.env` files,
- Google Cloud credential files,
- Application Default Credentials JSON,
- raw account screenshots without redaction,
- billing screenshots with private project/account details,
- virtual environments such as `.venv/`,
- `node_modules/`,
- `.ruff_cache/`,
- `.google-agents-cli/` local run folders,
- ADK local session databases under `.adk/`,
- `__pycache__/` and compiled Python files,
- raw terminal logs that expose account or project identifiers,
- full whitepaper PDFs or raw generated study notes.

Official source material is linked instead of duplicated.

---

## Why this matters

Good course documentation should be useful without becoming noisy or unsafe.

For Day 3, I kept the pieces that explain the work:

- what skills are,
- how they were installed,
- how they were tested,
- what the ADK workflow did,
- what failed,
- how it was fixed,
- and what source files are worth preserving.

Local build artifacts and private configuration do not belong in the public repo.
